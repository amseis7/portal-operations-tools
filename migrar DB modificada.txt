Para integrar columnas VT
Entendido. Dejamos la lógica de exportación compleja para más adelante y nos centramos en actualizar tu base de datos actual sin borrarla, para que acepte los nuevos datos de VirusTotal.

El "truco de la migración manual" consiste simplemente en ejecutar comandos SQL (ALTER TABLE) para agregar las columnas nuevas a la tabla ioc que ya existe.

Como son varias columnas (vt_reputation, vt_motores_json, vt_md5, etc.), hacer esto en una sola línea de comando es propenso a errores.

La forma más segura y profesional es crear un pequeño script de Python temporal que haga el trabajo sucio por ti.

Aquí tienes los pasos exactos.

Paso 1: Crear el Script de Migración (migrar_vt.py)
Crea un archivo nuevo en la raíz de tu proyecto (al lado de run.py) llamado migrar_vt.py y pega este código.

Este script se conecta a tu base de datos, intenta agregar las columnas una por una y te avisa si hubo éxito o si ya existían.

Python

import sqlite3
import os

# Detectar ruta de la DB (igual que en tu config)
BASE_DIR = os.path.abspath(os.path.dirname(__file__))
DB_PATH = os.path.join(BASE_DIR, 'instance', 'app.db')

# Las columnas nuevas que definimos en models.py
nuevas_columnas = [
    ("vt_last_check", "DATETIME"),
    ("vt_reputation", "INTEGER"),
    ("vt_permalink", "VARCHAR(255)"),
    ("vt_md5", "VARCHAR(32)"),
    ("vt_sha1", "VARCHAR(40)"),
    ("vt_sha256", "VARCHAR(64)"),
    ("vt_motores_json", "TEXT")
]

def migrar():
    print(f"--- Iniciando migración en: {DB_PATH} ---")
    
    if not os.path.exists(DB_PATH):
        print("❌ ERROR: No se encuentra el archivo de base de datos.")
        return

    conn = sqlite3.connect(DB_PATH)
    cursor = conn.cursor()

    for columna, tipo in nuevas_columnas:
        try:
            # Comando SQL para agregar columna
            query = f"ALTER TABLE ioc ADD COLUMN {columna} {tipo}"
            cursor.execute(query)
            print(f"✅ Columna agregada: {columna}")
        except sqlite3.OperationalError as e:
            if "duplicate column" in str(e):
                print(f"ℹ️  La columna '{columna}' ya existía. (Saltando)")
            else:
                print(f"❌ Error agregando '{columna}': {e}")

    conn.commit()
    conn.close()
    print("--- Migración Finalizada ---")

if __name__ == "__main__":
    migrar()
Paso 2: Ejecutar la Migración
Ahora tienes dos escenarios, elige el que corresponda a donde estás trabajando:

Escenario A: Estás en tu PC Local (PyCharm)
Simplemente ejecuta el script desde la terminal de PyCharm:

Bash

python migrar_vt.py
Si todo sale bien, verás varios ✅ Columna agregada.

Escenario B: Estás en Docker (Servidor)
Si quieres aplicar esto a tu servidor en producción sin borrar el volumen, debes hacer que el contenedor ejecute este script.

Asegúrate de que el archivo migrar_vt.py esté en tu carpeta del proyecto.

Reconstruye la imagen (para que el archivo migrar_vt.py se meta dentro del contenedor):

Bash

docker build -t portal-operations-tools:vt_beta .
Ejecuta el script dentro de un contenedor efímero (conectado a tus datos):

Bash

docker run --rm -v portal_db:/app/instance portal-operations-tools:vt_beta python migrar_vt.py
